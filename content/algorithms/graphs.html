Graphs

Description

A graph, \(G\) is a set of vertices and edges that are connected. Using the notation of Cormen et. al, \(G = (V, E)\). \(|V|\) and \(|E|\) denote the number of the respective elements. Graphs can be directed or undirected. That is, the possible direction of traversal between vertices can be specified or not.

Graph Traversal

Depth First Search (DFS)

For each vertex connected to the starting vertex (source), search along the path of connected vertices for as far as possible before moving to the next vertex connected to the source. Here is a generalized example for an undirected graph that returns all of the connected vertices:

class Graph {
    private adjacencyList: Map<number, number[]>;

    constructor() {
        this.adjacencyList = new Map();
    }

    addVertex(vertex: number): void {
        if (!this.adjacencyList.has(vertex)) {
            this.adjacencyList.set(vertex, []);
        }
    }

    addEdge(vertex1: number, vertex2: number): void {
        this.adjacencyList.get(vertex1)?.push(vertex2);
        this.adjacencyList.get(vertex2)?.push(vertex1);
    }

    dfs(start: number): number[] {
        const visited: Set<number> = new Set();
        const result: number[] = [];

        const dfsVisit = (vertex: number): void => {
            visited.add(vertex);
            result.push(vertex);

            const neighbors = this.adjacencyList.get(vertex);
            if (neighbors) {
                for (const neighbor of neighbors) {
                    if (!visited.has(neighbor)) {
                        dfsVisit(neighbor);
                    }
                }
            }
        };

        dfsVisit(start);
        return result;
    }
}

Run through/ Usage:

  • Initialize the graph: 
    const graph = new Graph()
  • Add some vertices. You can see in the addVertex function that these are just equivalent to numbers in the first level of keys in the adjacencyList: 
    graph.addVertex(1);

graph.addVertex(3);

  • Edges can then be added for any given vertex by assigning another vertex to the set of values in the adjacency list: 
    graph.addEdge(1,2);
graph.addEdge(1,3);
graph.addEdge(2,4);

    1: [2, 3], 
    2: [1, 4], 
    3: [1], 
    4: [2]

    An application: Convexity

    Convexity uses DFS to assess the completeness of a group of connected vertices.

    The game uses a graph class for each set of connected vertices ("dots") called a "cluster". When 3 or more dots are connected - i.e. the minimum for 2d shape completion - and a cycle is found it flags the shape as completed. If this happens, additional checks are performed to see if the completed shape is convex or not (see how convexity works).

    detectShapes() {
    const visited = new Set();
    this.completedShape = false;
    const dfs = (dot, startDot, path) => {
    
      // Base case for the recursion: 
      if (visited.has(dot)) {
        if (dot === startDot && path.length >= 3) {
          return path;
        }
        return null;
      }
      visited.add(dot);
      path.push(dot);
        
      // Recursion: 
      for (const neighbor of dot.connections) {
        const result = dfs(neighbor, startDot, path.slice());
        if (result) {
          return result;
        }
      }
      visited.delete(dot);
      return null;
    };

    for (const dot of this.dots) {
      const shape = dfs(dot, dot, []);
      if (shape) {
        this.completedShape = true;
        this.shape = shape;
        this.isConvex = this.checkConvexity();
        return shape;
      }
    }

    return null;
  }
